Intro

Manifesto

This is a guide to the available styles in the application. Here is a general outline and key of good things to know:

  1. Styles are mobile first — so consider making them lightweight and perform well
  2. Styles use a namespace — prefix them with: v-
  3. Styles are atomic — designed to handle a specific thing: -p for padding, -m for margin, -b for border, ...
  4. Styles are responsive — each style can be extended to target devices using a prefix of: -s- -m- -l-
  5. Styles are concise — by limiting variations using a common scale: -1 -2 -3 -4

Settings

The current settings used are:

  • v-<style>1 0.625rem or 10px
  • v-<style>2 1.25rem or 20px
  • v-<style>3 2.5rem or 40px
  • v-<style>4 3.75rem or 60px
  • v-s-<style> 35.5em and above or >= 568px
  • v-m-<style> 64em and above or >= 1024px
  • v-l-<style> 79.875em and above or >= 1278px

Gutters

The core of the grid is dependant on widths and floats, it is best to defer margins and paddings to the contents of the grid column — and this includes gutters.

v-clearfix v-mxn2
v-col-4
v-mx2

v-mx2
v-col-4
v-mx2

.v-mx2 — Provides a gutter margin to the contents with 2 spaces on the x axis (left/right)
.v-mxn2 — Provides a gutter margin to the grid container with negative 2 spaces on the x axis (left/right)

Responsive Grid

Responsive utility classes (v-s, v-m, v-l) are available as well to specify different column sizes at small (s), medium (m) and large (l) breakpoints to create responsive grids! Let's take a look:

v-col-12
v-s-col-6
v-m-col-4
v-col-12
v-s-col-6
v-m-col-4
v-col-12
v-s-col-6
v-m-col-4

.v-col-12 — Default breakpoint has 12 column width (full width)
.v-s-col-6 — Small breakpoint and above turns into 6 column width (half width)
.v-m-col-4 — Medium breakpoint and above turns into 4 column width (third width)

Note: Since there is no large breakpoint specified, the 4 column width still applies to large screens.
